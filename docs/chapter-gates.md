# Chapter Gates

A chapter is produced with mandatory gates to reduce drift.

## Gate 1: READ (mandatory)

Before outlining or drafting, read:

- `project-brief.md`
- `style-guide.md`
- `characters.md`
- `world-bible.md`
- `outline.md`
- `progress-tracking.md`

If it exists for this chapter:

- `chapter-outline/XXXX.md`

## Gate 2: PLAN (mandatory)

Create or update `chapter-outline/XXXX.md` with:

- Goal: what the protagonist wants in this chapter
- Conflict: what blocks it (person/system/rule)
- Action chain: what the protagonist does, step by step
- Cost: what is lost / revealed / changed
- New information: what the reader learns
- Emotional beat: where the tension rises and releases
- Hook: why the reader must open the next chapter

## Gate 3: WRITE + CHECK (mandatory)

Draft `chapters/XXXX.md`, then run self-checks:

- AI-smell check: repetition, template phrasing, vague abstractions
- Wateriness check: scenes without plot/character movement
- Consistency check: motives, timeline, power rules, names

Record results in `reviews/XXXX.review.md`.

## Gate 4: UPDATE MEMORY HUB (mandatory)

Append to `progress-tracking.md`:

- chapter summary (hard facts)
- new foreshadowing seeded
- foreshadowing resolved
- character deltas (relationships, stance, power, mindset)
- canon/setting changes
- next chapter focus (1-3 items)
- TODOs
